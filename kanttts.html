项目概述

项目背景

在中国电子科技集团公司第二十研究所实习期间，基于阿里KAN-TTS框架搭建语音合成训练环境，完成自定义声音模型的训练与优化，并搭建Flask接口提供端到端的语音合成服务。

项目实现了从环境配置、数据准备、模型训练到API服务部署的完整流程，展现了AI工程实践能力。

技术目标

• 构建GPU训练环境容器化部署
• 完成自定义声音数据的模型微调
• 提供RESTful API语音合成服务

技术实现

完整的模型训练与部署流程

1

Docker GPU 环境配置

基于 NVIDIA CUDA 12.1 + Ubuntu 20.04 构建训练容器，配置 Anaconda 环境与 ModelScope SDK。

docker run --gpus all -it nvidia/cuda:12.1.0-runtime-ubuntu20.04 conda create -n mytts python=3.7.13 pip install tensorflow==1.15.4 pip install modelscope==1.5.2
2

训练数据打标

使用 ModelScope 自动标注工具对原始语音数据进行预处理，生成训练所需的标注文件。

from modelscope.tools import run_auto_label input_wav = "/home/aiwu/KAN-TTS/test_female/" output_data = "./output_training_femaledata/" ret, report = run_auto_label(input_wav=input_wav, work_dir=output_data, resource_revision="v1.0.6")
3

SAMBERT 模型微调

基于预训练模型 speech_personal_sambert-hifigan_nsf_tts_zh-cn_pretrain_16k 进行微调训练。

train_info = { TtsTrainType.TRAIN_TYPE_SAMBERT: { 'train_steps': 202, 'save_interval_steps': 200, 'log_interval': 10 } } trainer = build_trainer(Trainers.speech_kantts_trainer, ...) trainer.train()
4

Flask API 部署

搭建 RESTful API 服务，提供文本到语音的合成接口，支持与业务系统集成。

@app.route("/tts", methods=["POST"]) def tts(): data = request.get_json() output = inference(input=data['data']) wav = output[OutputKeys.OUTPUT_WAV] return jsonify({'audio': wav}) app.run(host='0.0.0.0', port=8089)

工程挑战

问题排查与解决方案

🔧 sox 库缺失 已解决

数据打标过程中提示 sox 音频处理库未安装，导致标注脚本无法运行。

# 安装系统级音频处理工具 sudo apt-get -y update sudo apt-get install sox # 或通过 pip 安装 Python 绑定 pip install sox
🔧 typeguard 版本兼容 已解决

typeguard 3.0 版本中移除了 check_argument_types 函数，导致 ModelScope SDK 无法导入。

# 卸载新版本，安装兼容版本 pip uninstall typeguard pip install typeguard==2.13.3
🔧 KAN-TTS 模块缺失 已解决

训练脚本提示缺少 kantts 核心模块，需要额外安装阿里定制的 TTS 工具包。

# 从 ModelScope 官方源安装 pip install kantts -f https://modelscope.oss-cn-beijing.aliyuncs.com/releases/repo.html
🔧 依赖包冲突 已解决

训练过程中缺少多个依赖包：unidecode、pytorch_wavelets、pysptk 等。

# 安装所有必需依赖 pip install unidecode pip install pytorch_wavelets pywt tensorboardX pysptk bitstring
